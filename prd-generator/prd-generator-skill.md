# PRD Generator Skill

将用户提供的原始需求信息，转化为一份结构完整、可供 AI 直接执行的 PRD 文档。

---

## 工作流程

### Step 1：信息收集（必须先执行）

在生成 PRD 之前，**必须**通过提问确认以下信息。用户提供的信息越完整，生成质量越高。

将所有问题分为两类：

**【必问 - 没有无法生成】**

1. **功能/产品名称**：这个需求叫什么？
2. **要解决的核心问题**：用户/业务当前遇到什么痛点？
3. **目标用户**：谁会用这个功能？有哪些角色？各自权限范围是什么？
4. **核心功能描述**：需要实现哪几个主要功能点？（哪怕是一句话描述也行）
5. **技术上下文**：
   - 现有技术栈（前端框架/后端语言/数据库）
   - 有没有现成组件库或 UI 规范？
   - 有没有禁止引入的依赖？

**【选问 - 能显著提升质量】**

6. **明确不做的内容**：本期迭代有哪些功能是明确不在范围内的？
7. **性能/兼容性要求**：有没有具体的响应时间、并发量、浏览器兼容要求？
8. **关键数据字段**：核心实体有哪些已知字段？（如已有数据库表结构更好）
9. **参考/竞品**：有没有可参考的竞品或内部已有功能？
10. **上线时间**：大概的开发周期或上线截止日期？

如果用户一次性提供了大量信息（如需求文档草稿、会议记录），则从中提取上述信息，仅针对缺失项追问。

---

### Step 2：信息确认

整理用户提供的信息，以简短的结构化形式回显给用户确认：

```
我理解的需求如下，确认无误后我将生成 PRD：

- 功能名称：[XXX]
- 解决问题：[XXX]
- 目标用户：[角色1（权限X）/ 角色2（权限Y）]
- 核心功能：[F1 / F2 / F3]
- 技术栈：[前端XXX / 后端XXX / 数据库XXX]
- 不做：[XXX]
- 截止日期：[XXX]

有需要调整的地方吗？
```

---

### Step 3：生成 PRD

用户确认后，按以下模板生成完整 PRD。

**生成原则：**

- 所有字段、枚举值、接口参数必须明确定义，不允许写"由研发决定"
- 每个功能点必须包含：前置条件、输入参数、状态机逻辑、边界场景、验收标准
- 数据结构用 TypeScript Interface 表示
- 边界场景至少覆盖：空值、越权访问、网络失败、并发操作
- 验收标准（AC）写成 Checkbox 格式，可直接用于测试
- 如果用户未提供某项信息，用 `[待确认：XXX]` 标注，不要自行杜撰

## 质量检查清单

生成 PRD 后，在输出前自查以下项目：

- [ ] 每个功能点都有「边界与异常」章节
- [ ] 数据结构字段类型和枚举值已明确定义
- [ ] 状态机描述覆盖了成功路径和失败路径
- [ ] 验收标准可以直接作为测试用例执行
- [ ] 技术约束已写入「技术上下文」
- [ ] 权限矩阵已完整覆盖所有角色×功能组合
- [ ] 缺失信息已用 `[待确认：XXX]` 标注，而非自行填写

---

## 输出格式要求

- 输出为 Markdown 格式
- 代码块用对应语言标记（typescript / yaml / json）
- 文件名格式：`PRD-{功能名}-v1.0-{YYYYMMDD}.md`
- 生成完成后询问用户：「需要我针对某个功能点补充更多细节，或将此文档导出为 Word/PDF 吗？」
